## 2. 视频观察摘要

### 2.1 视频元数据

- 画幅：720 x 1280，竖屏 9:16
- 时长：约 27.17 秒
- 帧率：约 30fps
- 音频：AAC，44.1kHz，双声道，整体响度较高，存在连续背景音与高频战斗反馈

### 2.2 可见玩法

游戏场景为深空背景，中间是地球本体，外围出现大量金橙色外星飞行物、陨石或导弹状敌弹，从屏幕外圈向地球压迫。玩家侧武器来自地球内部、地表防御系统和近地轨道，小飞机从地球附近派出并向外拦截敌人，自动向外圈发射青蓝色子弹和白色轨迹攻击。击中后在环形边界产生白金色爆炸团，随着升级，地球派出的小型防卫机数量增多，火力明显提高。

视频出现 3 次 `Level Up` 升级界面。可见画面每次只展示三选一卡片，但正式游戏需扩展为随进度逐步解锁的升级池，至少具备 99 种可抽取选择。视频中的三张卡作为基础词条保留：

| 卡片 | 显示价格 | 视频中是否被选择 | 观察结果 |
|---|---:|---|---|
| Gun | $5 | 是，连续 3 次 | 选择后地球派出/强化近地小飞机，小飞机数量约从 3 增至 6，再增至 9 |
| Laser | $8 | 否 | 仅在卡面出现，效果需按名称与卡面推断 |
| Beam | $10 | 否 | 仅在卡面出现，效果需按名称与卡面推断 |

## 3. 产品目标

### 3.1 核心目标

还原一个“地球本体防守 + 外星环形弹幕 + 地球派出小飞机拦截 + 自动武器升级”的短局游戏。玩家在巨大压迫感的敌方包围中，通过升级地球防卫系统扩大火力圈，持续击碎靠近地球的敌人。

### 3.2 体验关键词

- 压迫：大量金橙色敌弹从外圈持续逼近中心。
- 爽感：青蓝色玩家火力持续扫射，边界爆炸不断连锁。
- 清晰：核心 UI 极少，升级时只看三张卡。
- 递进：每次升级后火力数量和屏幕反馈明显提升。

### 3.3 成功标准

- 第一眼能看出与视频同类：竖屏、中心地球、外圈金橙外星弹幕、青蓝地球防卫火力、三选一升级。
- 30 秒内至少触发 3 次升级，且连续选择 Gun 后地球防卫机数量达到 3/6/9 的可见递增。
- `Level Up` 不局限于 3 种固定选择，正式升级池至少包含 99 种可出现词条，并随局内阶段逐步扩容。
- 开发前必须先完成内置生图资产包，包括场景图、单位图、升级卡图、UI 面板、参考画面和图标。
- 在桌面浏览器和手机浏览器竖屏视口中保持画面完整，不出现 UI 遮挡。

## 4. 游戏模式

### 4.1 还原演示模式

用于验收“和视频一样”。该模式按视频节奏自动运行，并显示手形指针自动选择 Gun：

- 第 1 次升级：约 3.8s 弹出，选择 Gun。
- 第 2 次升级：约 11.8s 弹出，选择 Gun。
- 第 3 次升级：约 20.2s 弹出，选择 Gun。
- 27s 左右可继续循环或显示“继续战斗”状态，不做短视频水印复刻。

### 4.2 可玩模式

玩家可手动点击升级卡片。主体玩法仍为自动战斗：

- 玩家不直接操控星球移动。
- 武器自动锁定最近或最危险的敌弹。
- 玩家决策点集中在升级选择。
- 后续可扩展生命值、局外成长与更多关卡，但首版不进入范围。

## 5. 核心规则

### 5.1 胜负条件

视频未展示明确胜负界面。首版建议采用以下规则：

- 星球生命值：100。
- 敌弹触碰星球时造成伤害并消失。
- 生命值降至 0 时游戏结束。
- 生存 60 秒视为通关一局；还原演示模式不强制结束。

### 5.2 敌弹生成

敌弹是金橙色拖尾陨石/导弹状单位，沿屏幕外侧圆环生成，朝中心移动。

| 阶段 | 时间范围 | 敌弹密度 | 敌弹速度 | 目标体验 |
|---|---:|---:|---:|---|
| W1 | 0s - 4s | 中 | 慢 | 建立包围感，触发第一次升级 |
| W2 | 4s - 12s | 高 | 中 | 玩家 3 架战机能打出明显缺口 |
| W3 | 12s - 20s | 很高 | 中高 | 6 架战机压住局面，但仍有压迫 |
| W4 | 20s - 30s | 极高 | 高 | 9 架战机形成满屏青蓝弹幕和爆炸环 |

生成方式：

- 以星球中心为圆心，在屏幕外接圆附近按角度生成。
- 每波按扇区生成，让敌弹形成连续环形墙，而不是随机散点。
- 敌弹朝中心移动，但允许 3-8 度角度扰动，使拖尾自然交错。
- 敌弹进入星球半径范围即判定撞击。

### 5.3 玩家武器

#### 基础核心炮

开局存在星球基础防御，表现为白色双轨迹短弹：

- 发射频率：每 0.8 秒一次。
- 每次 2 发。
- 自动指向最近敌弹。
- 伤害较低，用于开局过渡。

#### Gun

视频中实际选择的升级。Gun 的效果是地球派出小型防卫机群，围绕地球近轨区域拦截外星来袭：

| Gun 等级 | 累计小飞机数 | 每架发射模式 | 视觉变化 |
|---:|---:|---|---|
| 0 | 0 | 无 | 仅基础核心炮 |
| 1 | 3 | 单发青蓝子弹，0.6s/次 | 地球周围出现三角小飞机 |
| 2 | 6 | 双发或快速单发，0.5s/次 | 火力圈更密，爆炸点增多 |
| 3 | 9 | 三发散射，0.45s/次 | 满屏青蓝弹幕，压制外圈 |
| 4+ | 每级 +3 | 提升射速、弹速、微散射 | 可用于长局扩展 |

小飞机行为：

- 从地球表面/近地轨道起飞，进入地球周围的防卫轨道。
- 围绕地球做圆周运动，半径约为地球半径的 1.8-2.6 倍。
- 每架小飞机始终朝向目标或朝外侧旋转。
- 发射青蓝色短能量弹，带发光拖尾。
- 小飞机不会离开地球防卫圈，不做画面外独立主角飞机。
- 小飞机不会被敌弹击毁，作为地球防卫系统的可视化火力单位存在。

#### Laser

视频中未展示实际效果，首版按卡面与名称实现：

- 生成 1 条持续 0.45 秒的白蓝色高亮激光。
- 优先瞄准最近星球的敌弹。
- 可穿透同一直线上的多个敌弹。
- 升级后增加持续时间、宽度和扫射角速度。
- 适合清理单方向压迫。

#### Beam

视频中未展示实际效果，首版按卡面与名称实现：

- 释放一段更宽的蓝白能量束，前摇 0.25 秒，爆发 0.7 秒。
- 范围比 Laser 更宽，冷却更长。
- 可清理扇形区域内敌弹。
- 升级后增加扇形角度、伤害和爆发次数。
- 适合作为高价强力清屏选择。

### 5.4 经济与升级

视频卡片显示价格但未展示余额。正式版本采用“隐藏资金 + 分阶段升级池 + 每次三选一”的方式，既保留视频里的简洁界面，又让后续选择越来越多。

- 每击毁 1 个普通敌弹获得 $1。
- 进入升级节点时暂停战斗，从当前已解锁升级池中抽取 3 张卡展示。
- 首次升级固定展示视频中的 `Gun $5`、`Laser $8`、`Beam $10`，用于还原原视频。
- 第二次升级开始，卡池逐步扩容，不再固定只展示三张基础卡。
- 若资金不足，卡片置灰；还原演示模式中保证 Gun 始终可选。
- 选择后扣除对应金额，播放确认动效，关闭面板继续战斗。

升级触发规则：

- 还原演示模式：按固定时间 3.8s、11.8s、20.2s 触发。
- 可玩模式：初版使用时间阈值触发，约每 8 秒一次；后续可改为经验条或击杀数触发。
- 每次升级只选 1 张卡。
- 若 6 秒内不选择，可自动选择当前可买的最低价卡，避免流程停死。

### 5.5 Level Up 选择池

#### 5.5.1 总体要求

- 升级池至少包含 99 种可抽取选择。
- 每次 `Level Up` 仍只展示 3 张卡，保持视频里的操作密度。
- 卡池随局内进度扩容：越往后，可抽取的升级越多、效果越复杂、稀有度越高。
- 每张升级卡都需要独立名称、图标/卡图、价格、稀有度、效果说明、音效标签和视觉反馈标签。
- 同一局中可重复抽到“可叠加升级”，但不可重复抽到“一次性解锁升级”。
- 升级卡必须全部服务“地球防御战”主题：地球小飞机、地表炮台、近地卫星、反导拦截、护盾、科研、资源调度和紧急指挥。
- 禁止把首版做成外部舰队、宇宙母舰、独立主角飞机或脱离地球的太空舰队作战。
- 升级卡不能只做数值换皮，至少要覆盖武器、弹道、防御、经济、爆炸、地球、近地卫星、特殊防卫技能、风险收益、连锁和终局强化 11 类玩法。

#### 5.5.2 解锁节奏

| 阶段 | 时间/升级次数 | 解锁池规模 | 抽卡规则 | 体验目标 |
|---|---:|---:|---|---|
| 初始还原 | 第 1 次升级 | 3 | 固定 Gun/Laser/Beam | 保留视频第一眼还原 |
| 早期 | 第 2-3 次升级 | 18 | 基础武器与基础数值强化 | 选择开始变多，但不复杂 |
| 中期 | 第 4-6 次升级 | 45 | 加入弹道、防御、爆炸、经济 | 形成 build 分支 |
| 后期 | 第 7-10 次升级 | 72 | 加入特殊技能、连锁、风险收益 | 每局开始产生差异 |
| 终局 | 第 11 次升级后 | 99+ | 全池抽取，稀有词条权重上升 | 提供长期可玩性 |

## 6. 画面与动效需求

### 6.1 画面层级

从后到前：

1. 深空背景：星点、暗色星云、轻微纵向视差。
2. 外星攻击外圈：金橙色拖尾敌方单位，大量重复生成。
3. 爆炸层：白金色粒子团、冲击环、短暂闪光。
4. 地球核心：地球质感球体，缓慢自转或光照移动。
5. 地球防卫层：从地球派出的近地小飞机、近地卫星、青蓝子弹、白色基础炮轨迹。
6. UI 层：Level Up 标题、卡片、手形指针、点击高亮。

### 6.2 关键动效

| 动效 | 触发 | 视觉描述 | 时长 |
|---|---|---|---:|
| 敌弹飞行 | 持续 | 金橙色长拖尾朝中心滑动，尾部带亮度衰减 | 持续 |
| 命中爆炸 | 玩家子弹命中敌弹 | 白金色小爆点 + 6-12 个粒子向外散开 | 0.25s |
| 边界连爆 | 多敌弹被击毁 | 爆点连接成弧状火焰边 | 0.4s |
| 小飞机起飞 | Gun 生效 | 小飞机从地球边缘/近地轨道出现并进入防卫环 | 0.45s |
| 小飞机巡航 | Gun 生效后 | 小飞机围绕地球旋转并微幅漂移 | 持续 |
| 子弹发射 | 小飞机开火 | 青蓝短弹带 8-16px 发光拖尾 | 0.3s - 0.8s |
| Level Up 入场 | 升级触发 | 背景暗化，标题从上方缩放淡入，卡片依次弹入 | 0.35s |

### 6.3 UI 规格

升级界面需保留视频中的英文 UI，以保证还原：

- 标题：`Level Up`
- 首次卡片：`Gun`、`Laser`、`Beam`
- 后续卡片：从 99+ 升级池中抽取，卡片名称可以是中文或英文，但同一风格内需统一。
- 价格：首次为 `$5`、`$8`、`$10`，后续按升级库配置显示。
- 风格：深青黑底、浅青描边、科幻切角、轻微发光。
- 选中态：紫色填充、紫色电弧边框、白色确认圆环。
- 大卡图必须提前生成，运行时只负责按配置加载，不临时拼凑主视觉。

其他非视频 UI，例如开始按钮、暂停按钮、失败/通关文案，默认使用简体中文。

## 7. 音效需求

所有音效使用内置合成音或程序化生成，不依赖外部音频素材。

### 7.1 声音分层

| 声音 | 触发 | 声音设计 | 实现方式 |
|---|---|---|---|
| 背景氛围 | 游戏开始后循环 | 低频太空氛围 + 轻微脉冲节奏 | Web Audio 振荡器 + 滤波 + 混响 |
| 敌弹呼啸 | 敌弹密集进入画面 | 轻微风切/下坠感，不抢主音量 | 噪声源 + 高通滤波 + 音量随机 |
| Gun 开火 | 战机发射 | 清脆短促的青蓝能量弹声 | 方波/三角波短包络 |
| 命中爆炸 | 敌弹被击毁 | 白金色爆点对应的短爆裂 | 噪声源 + 快速衰减 + 低频补强 |
| Level Up | 升级弹窗出现 | 上扬科幻提示音 | 正弦波滑音 + 延迟 |
| 卡片悬停 | 指针经过卡片 | 很短的电子点击 | 高频短包络 |
| 卡片确认 | 点击升级 | 紫色能量确认音，带上扬尾音 | 双振荡器 + 滤波扫频 |
| 生命受击 | 敌弹撞击星球 | 低沉冲击，屏幕轻震 | 噪声 + 低频正弦 |

### 7.2 混音要求

- 背景氛围音量低于战斗音效，避免持续压耳。
- 连续爆炸需要做声音合并，同 80ms 内多个爆点只播放 1-2 个爆炸音，避免过载。
- 移动端必须在首次点击后启动音频上下文，避免浏览器自动播放限制。
- 设置总音量开关，默认开启。

- 卡图风格：深青黑 UI 底、浅青描边、强光武器主体、清晰中心构图。

- 每个 99 升级词条都有可追踪的图像资产或图标资产。

- 所有项目引用的图片必须保存在工作区内，不能只留在默认生成目录或临时目录。
- 首版开发前至少完成一套完整可运行资产包；后续可以迭代替换，但不能用空白占位图进入开发验收。
- 运行时可继续使用程序化粒子增强爆炸、拖尾、光束和震屏，但主视觉素材必须来自提前生成的资产包。

## 9. 技术实现建议

首版建议做成浏览器竖屏 Web 游戏：

- 渲染：Canvas 2D，主循环使用 `requestAnimationFrame`。
- 音频：Web Audio API 合成音效，必要时用 `<audio>` 做兜底。
- 状态管理：轻量状态机，区分 `intro`、`playing`、`levelUp`、`gameOver`。
- 资产加载：开发前生成资产包，运行时基于 `manifest.json` 预加载背景、单位、升级卡图和 UI 图。
- 升级系统：配置驱动，99+ 词条从数据表读取，抽取权重和解锁阶段不写死在 UI 里。
- 本地开发端口：使用 46xx 命名空间，当前项目固定使用 `4647`，并已同步到桌面 `PROJECT` dashboard。

可行性依据：

- MDN `requestAnimationFrame` 文档说明动画可基于回调时间戳逐帧计算进度，适合游戏主循环。
- MDN Web Audio API 文档说明其通过音频上下文和音频节点进行模块化路由，适合动态音效。
- MDN 游戏音频指南指出 Web Audio API 适合游戏音频，可精确控制样本播放时机，并可用于动态音乐体验。
- OpenAI 图像生成文档说明可通过文本提示生成或编辑图像，并可配置尺寸、质量、格式和透明/非透明背景；本项目采用内置生图能力提前生成项目资产。

## 10. 验收标准

### 10.1 画面还原验收

- 竖屏 9:16，中心星球位置、大小和视频接近。
- 外圈金橙色敌弹数量足够形成“包围环”。
- 选择 Gun 后战机数量明显按 3/6/9 增加。
- 青蓝色玩家弹幕、白金色爆炸、金橙敌弹形成清晰色彩对比。
- 升级面板布局、标题、卡片名称、价格和选中动效接近视频。
- 升级卡图来自提前生成的资产包，不允许首版使用纯文字占位卡。

### 10.2 玩法验收

- 游戏启动后自动进入战斗。
- 30 秒内至少出现 3 次升级选择。
- Gun/Laser/Beam 均可点击并产生不同战斗效果。
- 升级池至少有 99 种配置项，且可按阶段逐步解锁。
- 任意一次 Level Up 从已解锁池中抽取 3 张，不再长期固定为 Gun/Laser/Beam。
- 敌弹可被击毁，命中星球会扣生命。
- 游戏不会因大量敌弹和粒子在 30 秒内明显卡顿。

### 10.3 资产验收

- 工作区内存在完整生成资产包和 `assets/generated/manifest.json`。
- 至少 99 张升级相关卡图或图标已生成并被 manifest 引用。
- 至少 8 张完整画面参考图已生成，覆盖开局、Level Up、Gun 强化、Laser、Beam、后期弹幕、濒危、胜负反馈。
- 所有核心单位和 UI 资产均来自内置生图引擎，不使用视频截图、水印图或外部未授权素材。

### 10.4 声音验收

- 有背景氛围音。
- Gun 开火、命中爆炸、Level Up、卡片确认均有独立音效。
- 连续爆炸时声音不会爆音。
- 可一键静音。

### 10.5 适配验收

- 桌面浏览器竖屏模拟 720 x 1280 正常显示。
- 手机宽度 390px 以上不遮挡核心 UI。
- 横屏访问时应保持居中竖屏画布或提示竖屏体验更佳。
